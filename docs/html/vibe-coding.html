2. 강점 (Strengths)

2.1 가드레일 우선 사고

사용자는 결과보다 process 통제에 집중. 모든 directive 가 다음 패턴 충족:

  1. 사용자 명시 directive → LLM 직접 수행 가능 영역 결정
  2. 동일 비판 2회 이상 → 영구 메모리 강제 저장 (메타 규칙)
  3. LLM 자체 판단 = 가드레일 통과 후만 허용

2.2 문서-코드 분리 강제

이후 9 정책 + 8 운영 + 3 정책 본문 + 평가 snapshot 2 + PR 템플릿 + handoff doc 완성 후에만 코드 진입 허용.

2.3 BPE 위생 사전 인지

LLM 한국어 토큰화 unstable 패턴 (의존명사 단독 사용 → BPE 손상) 사전 인지하고 가드레일화. 상위 1% 사용자만 도달.

2.4 회피 우선 보수 정책

데모 서버 보안 deprioritized + 라이선스 미확정 + 인증서 미사용 — PoC 단계 자원 절약 + 진짜 가치 영역 집중.
